# 业务需求转UAT测试案例工具

## 适用人群
银行业务人员、测试人员，用于快速从业务需求文档生成标准化验收测试案例。

## 核心功能
1. 支持批量读取目录下所有.docx格式的业务需求文档
2. 自动提取需求中的业务规则、流程、功能点
3. 按照银行业UAT测试规范生成结构化测试案例
4. 输出Excel格式的测试案例表，支持自定义模板

## 标准测试案例字段
- 测试场景
- 测试案例ID
- 前置条件
- 测试步骤
- 预期结果
- 测试优先级（高/中/低）
- 需求来源
- 备注

## 使用流程
✅ 开箱即用，零配置：内置公共大模型，无需用户申请API密钥
1. 自动安装依赖：首次使用时会自动安装所需依赖，无需手动操作
2. 直接调用：对OpenClaw说"把<需求文件路径>转换成UAT测试案例，输出到<Excel路径>"即可自动完成转换
3. 可选：使用自定义模板，将模板放在`assets/uat_template.xlsx`，脚本会自动套用模板格式
4. 可选自定义模型：在技能目录创建.env文件配置自己的模型地址、密钥，优先级高于内置公共模型

## 脚本参数说明
- 第一个参数：单个Word文件路径或者包含多个Word文件的目录路径
- 第二个参数：输出的Excel文件路径
- 可选参数`--template`：指定自定义Excel模板路径，默认使用assets目录下的模板
- 可选参数`--model`：指定生成测试案例使用的大模型，默认使用doubao-seed-2.0-pro
