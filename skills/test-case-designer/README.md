# Test Case Designer Skill - 测试案例设计技能

## 📁 目录结构

```
test-case-designer/
├── SKILL.md                          # 技能详细说明文档
├── README.md                         # 本文件 - 快速入门指南
└── 测试案例 - 企业标准模版.xlsx       # 企业标准测试案例模版（16 字段）
```

## 🎯 技能用途

本技能用于将测试需求分析结果转换为**企业标准格式**的结构化、可执行测试案例文档。

## 📋 企业标准模版

模版位置：`test-case-designer/测试案例 - 企业标准模版.xlsx`

### 16 个标准字段：

1. 功能模块名称
2. 测试案例名称
3. 所属系统
4. 关联需求名称
5. 案例性质
6. 测试概述
7. 步骤描述
8. 测试数据
9. 预期结果
10. 运行结果
11. 设计者
12. 资产编号
13. 是否纳入上线验证
14. 验证人
15. 计划验证日期
16. 案例类型

## 🚀 快速使用

### 示例 1：基于需求文档生成测试案例
```
请基于企业网银需求分析设计测试案例，输出为 Excel 格式，遵循企业标准模版
```

### 示例 2：针对特定模块
```
请针对客户强制移交模块设计测试案例，包含正案例和反案例
```

### 示例 3：基于缺陷分析
```
请针对信贷系统缺陷分析报告中的高风险模块设计回归测试案例
```

## 📤 输出格式

支持多种输出格式：
- ✅ **Excel** (.xlsx) - 企业标准格式，推荐用于正式交付
- ✅ **HTML** (.html) - 适合评审会议
- ✅ **CSV** (.csv) - 适合数据交换
- ✅ **Markdown** (.md) - 适合文档嵌入

- **test-case-review** - 测试案例评审技能（质量检查）
